# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do not open a public issue.**

Instead, email **security@amodalai.com** with:

- A description of the vulnerability
- Steps to reproduce the issue
- Any relevant logs or screenshots
- Your assessment of the severity

## What to Expect

- **Acknowledgment** within 48 hours of your report
- **Status update** within 5 business days with our assessment
- **Resolution target** of 90 days from confirmed vulnerability to fix

We follow coordinated disclosure. We ask that you give us reasonable time to address the issue before any public disclosure.

## Scope

The following are in scope for security reports:

- Authentication and authorization bypasses
- Injection vulnerabilities (SQL, command, etc.)
- Credential or secret exposure
- Remote code execution
- Cross-site scripting (XSS) in the chat widget or runtime app

The following are out of scope:

- Issues in dependencies (please report these upstream)
- Social engineering
- Denial of service (unless trivially exploitable)

## Recognition

We appreciate the security research community. Reporters of valid vulnerabilities will be credited in the release notes (unless they prefer to remain anonymous).
